# 调试日志与三次 API 拆分

每轮玩家选选项后，程序按 **串行** 调用最多 3 次 DeepSeek（`USE_SPLIT_FIRST = true` 时不用合并 JSON）。

## ① 角色回复 `拆分·①reply`

| 项 | 内容 |
|----|------|
| **目的** | 生成锋利本句台词（≤40 字） |
| **system** | 角色风格 + **完整** `plotSummary`（若有）+ 输出格式约束 |
| **user/messages** | 最近 **2 轮**对白（`HISTORY_TURNS`，约 ≤1400 字），不含 intent 标签 |
| **调试** | 黄 `→拆分·①reply`、绿 `←拆分·①reply`（含 system 内摘要全文） |

## ② 选项 `拆分·②选项`

| 项 | 内容 |
|----|------|
| **目的** | 生成深挖 / 推进两条选项 JSON（挂起由程序固定） |
| **system** | 选项规则（双选项 JSON schema） |
| **user** | 角色名、上一句、最近 **3 轮**对白（`OPTIONS_HISTORY_TURNS`）、**摘要摘录**（仅 `[待核实]` 段，≤400 字） |
| **时机** | 在 ① 的 reply **写入 session 之后**再调（options 能看到本轮 assistant） |
| **调试** | 黄 `→拆分·②选项`、绿 `←拆分·②选项` |

## ③ 剧情摘要 `拆分·③摘要`（仅第 4、8… 轮选项）

| 项 | 内容 |
|----|------|
| **目的** | 压缩长程记忆 `plotSummary`（≤1200 字，A+ 档案格式） |
| **system** | 摘要压缩规则（`SUMMARY_SYSTEM`） |
| **user** | 自检前缀 + 【已有摘要】+ 【新增对话】（保护区外旧对白 + 本轮完整对白） |
| **时机** | ①② 完成且 **assistant 已写入 session** 之后 |
| **调试** | 黄 `→拆分·③摘要`、绿 `←拆分·③摘要` |

下一轮起：① 的 system 带**完整**新摘要；② 的 user 带**待核实摘录**。

## 调试面板：必打 vs 不必打

### 必须保留（查 API 与轮次）

| 颜色 | 内容 |
|------|------|
| **黄 / 绿** | 三次 API 的 **完整请求 JSON / 返回正文**（唯一权威来源） |
| **灰** | `API 路径`（本轮回哪几步） |
| **灰** | `③摘要 · 即将请求` / `③摘要 · 已写入 session`（仅压缩轮，一行元数据） |
| **灰** | `摘要未执行` + 原因（非 4/8 轮或已压过等） |
| **灰** | `玩家选择` 等用户操作 |
| **橙 / 红** | JSON 修复、API 失败、摘要格式告警 |

### 不必打（已与黄绿重复或噪声大）

| 已移除 | 原因 |
|--------|------|
| 摘要·入模全文（灰） | 与黄条 `→拆分·③摘要` 的 `messages` 重复 |
| 摘要·写入 session 全文（灰） | 与绿条 `←拆分·③摘要` 重复 |
| 摘要·system 字数 | 无诊断价值 |
| 每轮剧情摘要缓存全文 | 已在 ① 黄条 system 可见 |
| 本轮 messages 字数 | 黄条 JSON 已含 |
| 选项已更新（界面展示） | 绿条 ② 已含 |
| 选项组装 | 固定逻辑，非 API |
| 重复的 API 路径（options-ai 内） | 由 app 每轮打一条 |

复制偏好：**默认勾选全部 API**；灰条可用「仅 API」快速导出三次请求。
