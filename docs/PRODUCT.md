# Points-of-mess 产品方向

本文档记录已确定的产品定义与原则，供后续实现对照。

## 是什么

**Points-of-mess** 是个人专用的静态 Web 应用：一片象征「混乱」（mess）的宇宙，其中分布着许多 **点（point）**。每个点由你创建，并写入 **性格 / 角色设定**；进入该点后，由 AI **按设定表演该角色**，与你对话。

- **点**：你创建的内容载体（位置、名称、性格设定等），不是玩家本人，也不是独立的「无主人 AI」。
- **玩家**：你，在宇宙中遨游、创建点、进入点对话。
- **AI**：在同一把 API 密钥下，根据该点的设定扮演角色；不同点靠 **不同的 system 设定 + 该点独立对话历史** 区分。

## 已确定的原则

| 项目 | 决定 |
|------|------|
| 使用范围 | **个人专用**，不依赖其他玩家 |
| 玩家间聊天 | **不做**（无 P2P） |
| 注册 / 登录 | **不需要** |
| 页面实时更新 | **不需要**（无多人同步；本地打开即为本机状态） |
| 运行方式 | **纯静态前端**，无需 Node；浏览器打开即可 |
| API 密钥 | **一个密钥** 服务所有点；角色差异由设定与上下文承担，不靠多密钥 |
| 部署 | 静态托管（如 GitHub Pages）；开发分支 `cursor/deepseek-chat-d862` |

## 对话与成本

- 聊天记录保存在 **本机**（按点分开）；模型 **不会**自动拥有「你没发出去」的内容。
- 每次请求发送：`该点的性格设定（system）` + `你选择携带的历史轮次` + `当前新消息`。
- **不必每次发送全部历史**；建议只带 **最近若干轮** 以控制 token 成本。更长记忆可用本地存档查阅，或通过日后「摘要」再送入请求（可选）。
- 清空某点的对话上下文时：可保留性格设定，仅不再把旧轮次发给 API。

## 目标体验（规划）

1. **宇宙层**：在 mess 宇宙中浏览、选择点（规划；当前原型为单页聊天）。
2. **点层**：创建点，填写性格设定（提示词 / 角色卡）。
3. **对话层**：进入点后，AI 扮演该点角色，流式回复，可停止生成、可清空该点对话。

品牌语气：从混乱中整理出可带走的 **要点**；宇宙是 mess，点是你落下的锚，进门后与 **角色** 对戏，而非与「另一个玩家」聊天。

## 当前实现状态

- 已实现：单点 DeepSeek 流式对话、停止 / 清空、`localStorage` 会话、静态页面模块化（`js/` + `styles/`）。
- 未实现：宇宙地图、多点创建与管理、按点拆分设定与历史。

## 技术结构（现行）

```
index.html
styles/chat.css
js/config.js    # API 密钥与全局配置
js/state.js     # 状态与本地存储
js/render.js
js/api.js       # 调用 DeepSeek
js/app.js
```

密钥位于 `js/config.js`，适用于个人自用；公开仓库需注意泄露风险。
