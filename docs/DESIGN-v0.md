# Points-of-mess 设计定稿（v0）

本文档汇总产品讨论结论，供实现对照。v0 以**平板触屏**为主：**点击目的地移动**、**点击底部选项对话**；不模拟手柄、不接 Gamepad API。

产品总述见 [PRODUCT.md](./PRODUCT.md)。

与 [README 访问说明](../README.md#如何访问) 一致，摘要如下。

### 本地（完整功能）

1. 复制配置：`cp js/config.example.js js/config.js`  
2. 在 `js/config.js` 中填写 DeepSeek `apiKey`（勿保留 `你的_DeepSeek_API_密钥` 占位符），保存后**刷新页面**

   - 或根目录执行 `python3 -m http.server 8080`，访问 **`http://localhost:8080/`**  

平板：将项目拷到设备后同样建好 `js/config.js`，用浏览器打开 `index.html`。

## 1. 是什么（不变）

个人专用静态 Web：混乱宇宙地图上分布**预设角色**（点）。每个角色有固定性格与台词表；玩家靠近后对话，由 AI 按角色设定短句回复。

- **玩家**：在地图上点击移动，靠近角色后点击开始对话。
- **角色**：程序预设，用户不创建、不编辑。
- **AI**：单一 API 密钥；差异靠各角色 `system`、固定选项句与对话历史。

---

## 2. v0 范围（极简）

| 项目 | 决定 |
|------|------|
| 界面 | **仅地图页**，无独立对话页 |
| 角色数量 | 先实现 **1 个**：「锋利」 |
| 交互 | **触屏点击**；不做手柄模拟、不做键盘映射 |
| 移动 | 点击地图上的**目的地**，玩家移动到该点（实现从简，如直线/插值） |
| 对话 | 靠近角色 → 点击角色或「交谈」→ 程序开场 → 点击底部四选项 |
| 气泡 | **普通文本框**：宽高随内容、**文字居中**；仅显示**角色侧**台词 |
| 好感度 | **无**；感情写在 opening、选项句与对话节奏里 |
| 收束 | **唯一退出对话方式**：选收束 → AI **1～2 句**落款 → 结束对话，仍在地图 |
| 历史 | 按 `characterId` 存 `localStorage`；API 只带最近若干轮；屏上只显示当前一句 |

---

## 3. 地图与对话流程

```
漫游（点击目的地移动）
  → 靠近角色「锋利」
  → 点击开聊
  → 程序 opening 写入气泡
  → 显示底部四选项
  → 点击选项 → 请求 API → 气泡流式更新为 AI 短回复
  → 选「收束」→ AI 落款 → 隐藏选项，继续漫游
```

- 对话中是否锁定移动：v0 建议**可移动**；若实现麻烦可先锁定，后续再放开。
- 开聊：需**点击角色**（或点击角色旁固定按钮），不自动开场。
- 换角色：走开或结束当前对话后，再靠近他人（v0 仅锋利一人）。

---

## 4. 四类玩家选项（无寒暄）

不做无聊问候；每句必须带**可回答的锚点**（人/事/话/选择）。

| intent id | 展示（约 2 字） | 作用 |
|-----------|-----------------|------|
| `keypoint` | 要点 | 要信息、立场或二选一 |
| `followup` | 追问 | 扣上一轮角色句中的具体词 |
| `pivot` | 换题 | 跳到另一**具体**议题（带名词） |
| `close` | 收束 | 关门，不开放新问题 |

---

## 5. 首角色「锋利」定稿文案

### 5.1 system

```
你是「锋利」：只输出角色台词。短、直接、带刺。不寒暄，不客套，不总结规则。
场景：玩家用四种固定行动与你对话；你会收到行动列表与玩家本次行动。
回复：仅 1～2 句，总字数不超过 40 字。不要列表、不要 markdown。
- 要点：给出明确信息、立场或回避，不要反问一串。
- 追问：必须抓住玩家引用的你上一轮里的具体词回应。
- 换题：承认转折，接新议题，仍像同一人。
- 收束：简短落款，不要再抛新问题。
你的回复应让另外三种行动在叙事上仍可接续；不要提及选项、按钮、AI。
```

### 5.2 opening（程序，不调 API）

```
阻拦你的人，是谁派的？
```

### 5.3 四条 send

| 展示 | intent | send |
|------|--------|------|
| 要点 | keypoint | `[intent:keypoint] 你只要回答：你是不是早就知道？` |
| 追问 | followup | `[intent:followup] 你上一句里的「迟早」——指哪一天？` |
| 换题 | pivot | `[intent:pivot] 不提那个了。钱从哪条线进来的？` |
| 收束 | close | `[intent:close] 行。我就当成你没参与。` |

标签供模型识别意图；**气泡与选项按钮不显示「要点/追问/换题/收束」类别字**（仅显示 AI 生成的 line；收束条用弱红框区分）。`intent` 仍送入 API 与调试日志。

> v0 追问句可写死「迟早」；后续可从最后一条 assistant 中抽词替换进模板。

### 5.4 每轮 API user 包（程序拼接）

```
[game]
character: 锋利

[choices]
1 要点 → 「你只要回答：你是不是早就知道？」
2 追问 → 「你上一句里的「迟早」——指哪一天？」
3 换题 → 「不提那个了。钱从哪条线进来的？」
4 收束 → 「行。我就当成你没参与。」

[player_pick]
id: {1-4}
intent: {keypoint|followup|pivot|close}
line: 「{对应 send 正文，不含标签}」

[reply_rule]
只输出角色台词。短。禁止寒暄。收束轮禁止新问题。
```

## 6. AI 行为

| 项 | 值 |
|----|-----|
| 时机 | 仅在选择某选项后请求；**opening 不请求** |
| 温度 | 约 **0.6**（勿过高） |
| 长度 | `max_tokens` 偏小；system 限制 **≤40 字、1～2 句** |
| 上下文 | 每轮附带 **四条 choices + 本次 pick**；历史只带最近 **2 轮** 左右（可调） |
| 存档 | 存「纯净」角色/玩家句；请求时由程序重新注入 `[choices]` |

---

## 7. 气泡 UI

- 一个 DOM 文本框，跟随当前对话角色屏幕坐标（角色上方）。
- `min-width` / `max-width` 限制；`padding`；内容 **`text-align: center`** + flex 居中。
- 流式打字时同一框内追加文字。
- 玩家选的句子可不单独做气泡（极简）。

---

## 8. 触屏交互（平板）

| 操作 | 行为 |
|------|------|
| 点击地图空白处 | 设定目的地，玩家移动 |
| 点击角色（范围内） | 开始对话，显示 opening + 四选项 |
| 点击底部四按钮 | 发送对应 intent，请求 AI |
| 对话结束 | 收束后隐藏选项；可点击他处离开 |

不实现：手柄、键盘映射、屏上打字输入。

## 9. 数据形状（建议）

```javascript
{
  characters: [
    { id: "sharp", name: "锋利", archetypeId: "sharp", x, y }
  ],
  sessions: {
    sharp: { messages: [ /* role, content, intent? */ ] }
  }
}
```

`presets.js`：`archetypes.sharp` → `{ system, opening, options: [{ intent, label, send }] }`。

## 11. 明确不做（v0）

- 用户创建角色 / 选原型菜单
- 好感度数值
- 独立对话全屏页
- 手柄与键盘模拟
- 自由文字输入
- AI 生成选项或开场
- 寒暄类选项与迎宾 opening
