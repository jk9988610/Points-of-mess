面向日常使用和安卓平板。技术细节见 [PAGES-SETUP.md](PAGES-SETUP.md)、故障见 [PAGES-FIX.md](PAGES-FIX.md)。

## 1. 程序在哪里运行？

本项目是 **纯静态网页**（HTML + CSS + JavaScript），**没有**在 GitHub 上跑游戏逻辑的服务器。

| 环节 | 在哪里 | 做什么 |
|------|--------|--------|
| **写代码 / 存仓库** | GitHub（`main` 分支） | 保存源码；合并后触发 **Pages 部署** |
| **打开页面** | **你的设备浏览器**（平板 Chrome 等） | 加载 HTML/JS/CSS，在本地执行 |
| **调 AI** | **你的浏览器** → DeepSeek API | 打开页后登录：账号 `jk9988610`，密钥填 DeepSeek API Key（可让浏览器记住，不在仓库里） |

可以记成：

- **GitHub** = 网盘 + 自动发布静态站  
- **平板/电脑** = 真正「玩游戏」的地方  
- **GitHub Pages** = 把同一套网页放到公网地址，方便平板用浏览器打开，**不是**在云端替你跑对话

**正式在线地址：** https://jk9988610.github.io/Points-of-mess/

## 2. 安卓平板会不会改坏 HTML / JS？

**不会。** 安卓系统不修改你的 HTML/JS 源码；平板只是用 **浏览器**（Chrome、Edge 等）像电脑一样执行它们。

需要注意的只是 **浏览器环境**，和是不是平板无关：

| 现象 | 原因 | 建议 |
|------|------|------|
| 版本号还是旧的 | 浏览器缓存了 `version.js` | 用 Chrome：**⋮ → 清除此网站的缓存**，或无痕标签重开 |
| 点选项没反应 | 未登录或密钥无效 | 先登录；密钥须为有效的 DeepSeek API Key |
| 地图点不准 | 触屏与点击共用一套逻辑 | 已支持 `touchstart`；尽量用 Chrome 全屏打开 |
| 刷新后对话没了 | 测试模式 `PERSIST_SESSIONS=false` | 正常；要长期记忆需改 `js/state.js`（开发项） |

本项目的 JS 使用标准 Web API（`fetch`、`canvas`、`localStorage`），在 **现代安卓浏览器** 上与桌面 Chrome 行为一致。

1. 在电脑改代码 → 合并到 `main` → Actions 部署  
2. 平板只开 Pages 链接，等 1～3 分钟后强刷

## 4. 开发发布流程（给「写程序」时对照）

```
在 Cursor / 电脑改代码
    → push 到 GitHub（main 或 PR 合并）
    → Actions 部署 Pages
    → 平板浏览器打开 github.io 链接（强刷看新版本）
```

- **日常改玩法、密钥、调试**：优先 **本地** `index.html` + `js/config.js`  
- **给别人看页面、平板免拷文件**：用 **GitHub Pages**  
- 改版本号时同步改 `js/version.js` 和 `index.html` 里 `?v=`，减少平板缓存旧脚本

## 5. 经验反思（Pages 版本卡住）

这次 **分支已是新版、平板仍显示 0.2.5** 的典型原因：

1. **`gh-pages` 文件更新了，但 Pages CDN 没用上**  
   - 对比：raw 上 `version.js` 已是 0.2.8+，而 `github.io` 仍是 0.2.5  
2. **曾把 `.github` 带进 `gh-pages`**，导致 Jekyll **build 失败**，线上一直用**上一次成功**的旧包  
3. **只改 Settings 或只推分支**，没在 **Deployments** 里看到新的 **Success**  
4. **平板/浏览器缓存** `version.js`，误以为没部署成功  

**已采取的仓库侧改进：**

- 部署前打包 `_site`，**排除 `.github`**  
- `gh-pages` 使用 **`force_orphan: true`**，避免脏文件残留  
- 增加 **Republish Pages (official)** 一键重发  
- 根目录 **`DEPLOY.txt`** + 版本号，便于验收  

**你以后自检三步：**

1. Settings → Pages → Source = **GitHub Actions**  
2. Actions → **Republish Pages (official)** → Run → Deployments **Success**  
3. 平板无痕打开：`.../js/version.js` 与 `.../DEPLOY.txt`  

更细排错：[PAGES-FIX.md](PAGES-FIX.md)

## 6. 密钥与安全

- 仓库里的 `js/config.js` **不含** API 密钥；登录后密钥只存在本机 `localStorage` / `sessionStorage`  
- 账号固定为 `jk9988610`（假门禁）；「密钥」填你的 DeepSeek API Key，由浏览器密码管理器记住即可  
- **若曾在旧版把密钥提交到 GitHub/Pages，请立刻在 DeepSeek 控制台作废并换新密钥**

## 7. 对话与记忆（和 Cursor 聊天的区别）

- **本游戏**：每次请求内容由页面里的 JS 组装（选项路径带 `messages` 历史；输入框路径只发一条 `user`）
