# 安卓平板 · 从零开始（照着做即可）

你无法在「文件」里直接点开网页来聊天，需要先用 **Termux**（平板里的「终端」）启动服务，再用 **Chrome** 打开一个网址。

下面每一步都写了「在哪里点、输入什么」。

---

## 第 0 步：你要准备的东西

- 安卓平板已联网（Wi‑Fi）
- 整个项目文件夹 `Points-of-mess` 已在平板上  
  - 若还没有：见本文末尾 **「还没有项目文件夹？」**
- 你的 DeepSeek API 密钥（填进 `.env` 用）

---

## 第 1 步：安装 Termux（这就是「平板终端」）

1. 在平板浏览器打开：**https://f-droid.org/zh_Hans/packages/com.termux/**
2. 下载并安装 **Termux**（推荐 F-Droid，不要用已下架的 Play 商店旧版）。
3. 安装完成后，在桌面或应用列表找到 **Termux**，图标一般是黑色底。

> **Termux 是什么？**  
> 就是一个可以输入文字命令的黑窗口，相当于电脑上的「命令提示符」。**以后说的「打开终端」= 打开 Termux 这个 App。**

---

## 第 2 步：第一次打开 Termux

1. 点击打开 **Termux**。
2. 你会看到黑色屏幕、底部有键盘，可能有一行类似：
   ```text
   ~/ $
   ```
   这就是终端，**下面所有命令都在这里输入**。

3. （建议做一次）输入下面命令后按键盘上的 **回车**（Enter）：

```bash
termux-setup-storage
```

若弹出权限请求，点 **允许**。  
这样 Termux 才能读取你「下载」文件夹里的项目。

---

## 第 3 步：进入你的项目文件夹

项目如果在 **「下载」** 里，一般路径如下。  
**请一行一行复制、粘贴到 Termux，每行后按回车：**

```bash
cd ~/storage/shared/Download/Points-of-mess
```

如果提示 `No such file or directory`（没有那个目录），说明文件夹不在这个位置，改用下面命令**查找**：

```bash
ls ~/storage/shared/Download
```

看列表里有没有 `Points-of-mess` 或类似名字。  
若在别的位置，例如 `Documents`：

```bash
cd ~/storage/shared/Documents/Points-of-mess
```

**还不确定路径时**，在 Termux 执行：

```bash
find ~/storage/shared -maxdepth 3 -type d -name "Points-of-mess" 2>/dev/null
```

它会打印出完整路径，然后：

```bash
cd 这里换成上面打印出来的路径
```

成功进入后，再执行：

```bash
ls
```

应能看到 `index.html`、`server.js`、`pad-start.sh` 等文件。

---

## 第 4 步：安装 Node（只需做一次）

仍在项目目录里，复制粘贴：

```bash
pkg update -y
pkg install -y nodejs git
```

等待结束（可能要几分钟）。  
检查是否成功：

```bash
node -v
```

若显示版本号（如 `v20.x.x`）即可。

---

## 第 5 步：配置 API 密钥

**若已有 `.env` 且已填好密钥，可跳过到第 6 步。**

```bash
cp .env.example .env
```

然后用平板上的文本编辑器打开项目里的 **`.env`** 文件，改成：

```env
DEEPSEEK_API_KEY=你的真实密钥
```

保存后回到 Termux。

（也可以在 Termux 里用 nano 编辑：）

```bash
nano .env
```

改完后：按 **音量下键 + E**（或看屏幕提示保存），再 **Ctrl+X** 退出（Termux 里 Ctrl 用屏幕上的 **CTRL** 键配合 **X**）。

---

## 第 6 步：一键启动项目

在项目目录执行：

```bash
sh pad-start.sh
```

或：

```bash
npm run pad
```

第一次会下载依赖，请耐心等待。  
看到类似下面内容就表示成功：

```text
Chat server listening on 0.0.0.0:3000
【Pad 测试】请在 Safari 打开
  → http://localhost:3000
```

**启动成功后，不要关闭 Termux**（可切到后台，但不要划掉结束进程）。

---

## 第 7 步：用 Chrome 打开聊天页面

1. 打开 **Chrome**（或系统自带浏览器）。
2. 地址栏输入（可长按粘贴）：

```text
http://127.0.0.1:3000
```

3. 应出现「AI 智能对话」页面，即可测试发消息。

**建议：** Chrome 菜单 → **添加到主屏幕**，以后像 App 一样点开。

---

## 以后每次测试（只需 2 步）

1. 打开 **Termux** → 进入项目（若已在别的目录，再执行一次 `cd .../Points-of-mess`）→ 运行：

```bash
sh pad-start.sh
```

2. 打开 **Chrome** → 访问 `http://127.0.0.1:3000`

要停止服务：回到 Termux，按 **Ctrl+C**（先点屏幕 **CTRL**，再点 **C**）。

---

## 常见问题

| 问题 | 解决办法 |
|------|----------|
| 找不到 Termux | 从 F-Droid 安装，应用名就叫 **Termux** |
| `pkg: command not found` | 说明没装在 Termux 里，请打开 **Termux** 这个 App 再输入命令 |
| `cd` 失败 | 用第 3 步的 `find` 命令找文件夹真实路径 |
| 浏览器打不开网页 | 确认 Termux 里服务还在跑、没有报错；地址必须是 `http://127.0.0.1:3000` |
| 提示未配置 API Key | 检查 `.env` 里 `DEEPSEEK_API_KEY=` 后面是否有密钥 |
| 输入中文命令困难 | 命令全是英文，可从本文复制粘贴 |

更详细的说明：[docs/安卓平板-Termux手把手.md](docs/安卓平板-Termux手把手.md)

## 还没有项目文件夹？

**方式 A：GitHub 下载 ZIP（最简单）**

1. 平板浏览器打开你的仓库页面（或让朋友在电脑上传到你的网盘）。
2. 下载 ZIP，解压到 **「下载」** 文件夹，得到 `Points-of-mess` 文件夹。
3. 从本文 **第 3 步** 继续。

**方式 B：在 Termux 里用 git 克隆**

```bash
pkg install -y git
cd ~
git clone https://github.com/jk9988610/Points-of-mess.git
cd Points-of-mess
```

然后从 **第 4 步** 继续。
